PROJECT: QuitReady Report Layout + PDF Alignment Overhaul

Goal:
Fix the QuitReady report so the on-screen report and generated PDF are visually identical, professionally designed, and free of layout bugs. Implement a clean, unified layout system that eliminates drift between report and PDF rendering. This update should address cover page design, typography consistency, page 1 layout, encoding issues, spacing, and table styling.

CRITICAL ARCHITECTURE CHANGE

There must be ONE source of truth for the report layout.

Current problem:
HTML report layout and PDF layout appear to be separate templates. This causes visual drift and prevents updates from propagating.

Fix:

Create a single ReportLayout component used for both:

Report view
PDF export

Example structure:

/components/report/ReportLayout
/components/report/pages/*
/lib/report/renderPDF

Flow should be:

ReportLayout
↓
render in browser
↓
renderToPDF(ReportLayout)

Remove any separate PDF template system.

Both outputs must render the exact same layout components.
